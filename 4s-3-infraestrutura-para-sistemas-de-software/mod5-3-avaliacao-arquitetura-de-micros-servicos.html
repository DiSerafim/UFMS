Questionário ✅ ❌

Pergunta 1

O provedor de serviços na nuvem computacional não oferece "Container como um Serviço" (CaaS).

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Falso. ✅

Resolução:

  • Os provedores de serviços em nuvem frequentemente oferecem "Container como um Serviço" (CaaS) como parte de suas ofertas. O CaaS é uma solução que permite aos usuários implantar, gerenciar e escalar contêineres na nuvem, oferecendo a infraestrutura e as ferramentas necessárias para facilitar o gerenciamento de contêineres. Provedores populares como Google Cloud, AWS e Azure oferecem serviços de CaaS.

Pergunta 3

A tecnologia de Container tem as seguintes características, exceto:

  • a. Permite o escalonamento de aplicações.
  • b. Não usam bibliotecas do kernel do SO.
  • c. Torna processos de implantação e configuração mais simples.
  • d. O ambiente torna-se portável.
  • e. Simplifica a metodologia DevOps facilitando o desenvolvimento ágil.
Resposta: b. ✅

Resolução:

  • Os contêineres, na verdade, compartilham o kernel do sistema operacional do host. Eles utilizam as bibliotecas e recursos do kernel, mas operam de maneira isolada em relação aos processos e sistemas de arquivos.

Pergunta 4

Vantagens do Docker® como tecnologia de virtualização por container são, exceto:

  • a. Controle mais granular dos recursos, gerando mais eficiência da infraestrutura.
  • b. Os arquivos do container não podem ser compartilhados entre o host e o Container.
  • c. Possibilita a implantação de aplicações de forma mais ágil, confiável e estável, em qualquer ambiente.
  • d. Permite a replicação e padronização da aplicação.
  • e. Tecnologia Open-Source.
Resposta: b. ✅

Resolução:

  • Os contêineres podem compartilhar arquivos com o sistema operacional host por meio de volumes e montagens de diretório. Isso é uma característica importante da tecnologia de contêiner, permitindo que os dados persistam e sejam acessíveis entre o contêiner e o host.

Pergunta 5

Na plataforma Docker®, o pacote do software Kubernetes® permite o gerenciamento de vários containers Docker®.

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Verdadeiro. ✅

Resolução:

  • O Kubernetes é uma plataforma de orquestração de contêineres que permite o gerenciamento de múltiplos contêineres Docker, facilitando a automação de implantações, escalonamento e gerenciamento de aplicações em contêineres. Ele permite que você implemente, escale e gerencie aplicações em contêineres de forma eficaz em ambientes de produção.

Pergunta 7

Virtualização por Container é uma tecnologia que permite a criação de uma única instância isolada de um determinado sistema operacional dentro de um único hospedeiro.

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Falso. ✅

Resolução:

  • A virtualização por contêiner permite a criação de múltiplas instâncias isoladas de aplicações ou serviços, não apenas uma única instância. Cada contêiner pode rodar diferentes aplicações, e todos compartilham o mesmo kernel do sistema operacional do host, mas são isolados entre si.

Pergunta 8

A plataforma Docker® possui as seguintes versões, exceto:

  • a. Docker Business®
  • b. Docker Pro®
  • c. Docker Personal®
  • d. Docker Scout®
  • e. Docker Team®
Resposta: d. Docker Scout®. ✅

Resolução:

  • O Docker Scout não é uma versão, mas sim uma ferramenta ou funcionalidade que pode ser utilizada dentro do ecossistema Docker para análise e gerenciamento de segurança de imagens de contêineres.

Pergunta 10

Container como um Serviço (CaaS) é uma plataforma de transmissão de dados baseada em um drive de computador, perfeita para gerenciar seus containers.

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Falso. ✅

Resolução:

  • Container como um Serviço (CaaS) não é uma plataforma de transmissão de dados baseada em um drive de computador. Na verdade, CaaS é uma plataforma de hospedagem baseada em nuvem projetada especificamente para gerenciar e orquestrar contêineres. Ele permite que os desenvolvedores implantem, gerenciem e escalem aplicativos em contêineres em um ambiente de nuvem.
