Pergunta 1

Assinale a alternativa correta quanto à execução do comando a seguir no MongoDB.

                db.funcionario.find({
                    salario:{$gte:1000, $lte:20000}, $ and [{sexo:"Masculino"}]
                })
  • a. O comando remove os funcionários do sexo Masculino cujos salários são maiores ou iguais a 1000 ou menores ou iguais a 20000.
  • b. O comando retorna apenas os funcionários do sexo Masculino cujos salários são maiores ou iguais a 1000 e menores ou iguais a 20000.
  • c. O comando remove os funcionários do sexo Masculino cujos salários são maiores ou iguais a 1000 e menores ou iguais a 20000.
  • d. O comando atualiza os salários dos funcionários do sexo Masculino.
  • e. O comando retorna apenas os funcionários do sexo Masculino cujos salários são maiores que 1000 e menores que 20000.
  • f. O comando retorna todos os funcionários do sexo Masculino com salário de 1000 ou 20000.
Resposta: b. ✅

Resolução:

  • O comando, está buscando documentos na collection funcionario que atendem às seguintes condições:
    • salario: maior ou igual a 1000 ($gte: 1000) e menor ou igual a 20000 ($lte: 20000).
    • sexo: igual a "Masculino".

Pergunta 2

Os comandos a seguir estão corretos e são equivalentes.

                MongoDB:
                db.cliente.find({
                    nome:{$and:["Margarida","Ana"]}
                })

                SGBD relacional (SQL):
                SELECT * FROM cliente WHERE nome IN ('Margarida', 'Ana');

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Verdadeiro. ✅

Resolução:

$and em MongoDB não funciona dessa maneira. A forma correta de usar $and seria algo como:

                db.cliente.find({ $and: [{ nome: "Margarida" }, { nome: "Ana" }] })

O comando SQL está correto.

Pergunta 4

O MongoDB é um NoSQL chave-valor de alto desempenho e disponibilidade.

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Falso. ✅

Resolução:

  • O MongoDB é um banco de dados NoSQL, mas não é do tipo chave-valor. Ele é classificado como um banco de dados orientado a documentos. No MongoDB, os dados são armazenados em documentos BSON (uma extensão binária de JSON), que permite uma estrutura mais complexa e flexível do que o simples par chave-valor.

Pergunta 5

Assinale a alternativa CORRETA acerca do seguinte comando realizado no MongoDB:

                db.cliente.update(
                    {"nome":"Margarida"},
                    {$set:{"nome":"Margarida Teixeira"}},
                    {upsert:true}
                )
  • a. Se a condição de busca não for satisfeita, ou seja, o documento não for encontrado, o documento será inserido na coleção.
  • b. O comando apenas insere um novo documento na collection cliente.
  • c. O comando pode remover todos os clientes que possuem nome "Margarida Teixeira".
  • d. O comando busca por todos os clientes que possuem nome "Margarida Teixeira" e altera o atributo nome para "Margarida".
Resposta: a. ✅

Resolução:

  • O comando update no MongoDB, com a opção upsert: true, realiza uma atualização ou inserção (update or insert). Especificamente:
    • {"nome":"Margarida"}: Condição de busca. MongoDB procura documentos que correspondam a essa condição.
    • {$set: {"nome":"Margarida Teixeira"}}: Atualização. Se um documento correspondente for encontrado, o campo nome é atualizado para "Margarida Teixeira".
    • {upsert: true}: Opção de upsert. Se nenhum documento correspondente for encontrado, um novo documento é inserido com os valores especificados na condição de busca e na atualização combinados.

Pergunta 6

O termo NoSQL refere-se a:

  • a. São bancos de dados que não utilizam a SQL para garantir as propriedades ACID.
  • b. Implementação simultânea das três propriedades presentes no teorema CAP.
  • c. São bancos de dados que não utilizam a SQL, são distribuídos, possuem tabelas e esquema definido.
  • d. É uma abordagem teórica normalizada, tem como premissa a não utilização do SQL em bancos de dados heterogêneos.
  • e. São bancos de dados não relacionais, distribuídos e, em geral, de código aberto.
Resposta: e. ✅

Resolução:

  • O termo NoSQL (ou "Not Only SQL") refere-se a uma categoria de sistemas de gerenciamento de banco de dados que são não relacionais, distribuídos e geralmente de código aberto. Estes bancos de dados são projetados para grandes volumes de dados, diversidade de dados e a necessidade de operações rápidas e escalabilidade horizontal. Embora não utilizem SQL como linguagem principal de consulta, alguns NoSQL DBMS podem suportar SQL ou linguagens similares a SQL para certas operações. Eles são utilizados frequentemente em aplicações onde a flexibilidade, a performance e a escalabilidade são mais importantes do que a rigidez dos bancos de dados relacionais tradicionais.

Pergunta 7

O teorema CAP foi desenvolvido para descrever o comportamento de sistemas distribuídos. Esse teorema define três propriedades (consistência, disponibilidade e tolerância a partição), sendo que é impossível garantir essas três propriedades ao mesmo tempo.

Escolha uma opção: Verdadeiro ou Falso.

Resposta: Verdadeiro. ✅

Resolução:

  • O teorema CAP, proposto por Eric Brewer, estabelece que em um sistema distribuído é impossível garantir simultaneamente todas as três propriedades: Consistência (Consistency), Disponibilidade (Availability) e Tolerância a Partição (Partition Tolerance). Em outras palavras, um sistema pode, no máximo, garantir duas dessas propriedades ao mesmo tempo, mas não todas as três. Quando ocorre uma partição de rede (falha de comunicação), o sistema deve escolher entre ser consistente (garantir que todas as partes do sistema vejam os mesmos dados ao mesmo tempo) ou disponível (garantir que o sistema continue operando e respondendo a solicitações).

Pergunta 8

Assinale a alternativa que correlaciona corretamente os SGBDs NoSQL e seus modelos:

  • a. MongoDB: modelo orientado a colunas; Cassandra: modelo baseado em grafos; Neo4J: modelo orientado a documentos; Redis: Modelo chave-valor;
  • b. Cassandra: modelo orientado a colunas; Neo4J: modelo baseado em grafos; Redis: modelo orientado a documentos; MongoDB: Modelo chave-valor;
  • c. Redis: modelo orientado a colunas; Cassandra: modelo baseado em grafos; MongoDB: modelo orientado a documentos; Neo4J: Modelo chave-valor;
  • d. Cassandra: modelo orientado a colunas; Neo4J: modelo baseado em grafos; MongoDB: modelo orientado a documentos; Redis: Modelo chave-valor;
  • e. Redis: modelo orientado a colunas; Neo4J: modelo baseado em grafos; MongoDB: modelo orientado a documentos; Cassandra: Modelo chave-valor;
Resposta: d. ✅

Resolução:

  • Cassandra: é um SGBD NoSQL que utiliza um modelo orientado a colunas, ideal para grandes volumes de dados e alta disponibilidade.
  • Neo4J: é um banco de dados NoSQL especializado em grafos, permitindo a modelagem e consulta de dados altamente conectados.
  • MongoDB: é um SGBD NoSQL orientado a documentos, armazenando dados em formato BSON, que é uma forma binária do JSON.
  • Redis: é um banco de dados NoSQL chave-valor de alto desempenho, utilizado frequentemente para caching, sessões, e filas.
